# 動画リスト機能改善 要求内容

## 概要

動画リストの使い勝手を改善する。URLだけでは動画を識別しづらいため、タイトルとサムネイルを自動取得して表示する。また動画が増えすぎないよう上限を設ける。

## 実装する機能

### 1. 動画情報の自動取得
- 動画URL登録時に、タイトルとサムネイル画像を自動取得してローカルに保存する
- YouTube: oEmbed API でタイトル・サムネイルを取得する
- Instagram: HTMLのOGPタグからタイトル・サムネイルの取得を試みる。失敗した場合はドメイン名をタイトル、Instagramアイコンをサムネイル代わりに表示する
- 取得はURL登録時の1回のみ（以降はローカルデータを使用）

### 2. 動画リストの表示改善
- 動画セルにサムネイル画像とタイトルを表示する
- サムネイル取得中はローディング表示を出す
- 取得失敗時はプラットフォームアイコン（カメラ / 再生ボタン）で代替表示する

### 3. 動画登録数の上限
- 登録できる動画は最大10件までとする
- 10件に達したら＋ボタンを無効化し、上限に達した旨のメッセージを表示する

## ユーザーストーリー

- 動画リストを見たとき、サムネイルとタイトルで目的の動画をすぐ見つけられる
- 動画を10件登録しようとすると、それ以上追加できないことがわかる

## 受け入れ条件

- [ ] YouTube URLを登録するとタイトルとサムネイルが表示される
- [ ] Instagram URLを登録すると、取得できた場合はタイトル・サムネイルが、失敗した場合は代替表示が出る
- [ ] 動画セルにサムネイルとタイトルが表示される
- [ ] 動画が10件のとき＋ボタンが無効化される
- [ ] 動画が10件のときに上限メッセージが表示される
- [ ] アプリ再起動後もサムネイル・タイトルが保持される

## 制約事項

- 情報取得はURL登録時のみネットワークを使用する（以降はオフライン動作）
- 外部ライブラリは使用しない
